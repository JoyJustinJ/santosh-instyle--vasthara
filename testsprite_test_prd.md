# Test PRD: Vastra Application

## 1. Objective
To thoroughly test the frontend application of the Vastra CRM using TestSprite's automated E2E testing framework. This PRD focuses on validating the core user flows, authentication, and staff/admin operations.

## 2. Scope of Testing
The automated tests will cover the following critical areas:
- **Authentication:** Phone number login, OTP verification, PIN login, and Biometric login.
- **Customer Portal:** Scheme viewing, payment history, and profile access.
- **Staff Operations:** Enrolling new customers, recording installments, and initiating scheme fulfillments.
- **Admin Oversight:** Creating new schemes, exporting data (CSV/PDF), and managing staff authorities.

## 3. Test Cases (Requirement Groups)

### Requirement 1: User Authentication & Security
- **TC-1.1:** Verify that a user can request an OTP using a valid Indian phone number (+91).
- **TC-1.2:** Verify that entering a valid OTP successfully logs the user into the appropriate dashboard based on their role.
- **TC-1.3:** Verify that entering an invalid OTP displays an appropriate error message.
- **TC-1.4:** Verify that PIN and Biometric login options function correctly for returning users.

### Requirement 2: Customer Scheme Management
- **TC-2.1:** Verify that a logged-in customer can view their active scheme details.
- **TC-2.2:** Verify that the installment timeline accurately reflects paid vs. pending months.

### Requirement 3: Staff Workflows
- **TC-3.1:** Verify that a Staff member can successfully enroll a new customer, and that the customer is tagged with the Staff's Employee ID.
- **TC-3.2:** Verify that a Staff member can log a monthly installment payment.
- **TC-3.3:** Verify that scheme fulfillment requests trigger the OTP flow for the customer.

### Requirement 4: Admin Features
- **TC-4.1:** Verify that the Admin can download the Staff Report CSV.
- **TC-4.2:** Verify that the Admin can update a Staff member's access level (e.g., Standard to Manager).
- **TC-4.3:** Verify that PDF generation for receipts respects A4 pagination and doesn't cut text in half.

## 4. Execution Environment
- **Framework:** TestSprite Automated E2E
- **Target URL:** Local preview server (`http://localhost:4173`)
- **Mode:** Production build simulation

## 5. Expected Deliverables
- Raw Markdown Test Report (`raw_report.md`) generated by TestSprite.
- Completed, standardized Test Report categorizing passed/failed tests into the requirement groups above.
